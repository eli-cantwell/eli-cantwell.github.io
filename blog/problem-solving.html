Problem Solving

As someone who is new to coding, encountering problems while learning how to code is inevitable. One of the first problems I encountered was understanding the order of declared variables:

let nameOne = 'Marama'

let nameTwo = 'Wiremu'

let nameThree = nameTwo

nameTwo = nameOne

nameOne = nameThree

My first thought was that nameOne would have the value of 'Marama' because nameTwo = nameOne would set nameTwo's value to 'Marama’. I tried for a while to figure out why ‘nameOne’s’ value is Wiremu, and eventually asked one of my classmates if the reason was because ‘let nameThree = nameTwo’ was declared before reassignment of ‘nameTwo = nameOne’. He let me know that I was correct, and I learned a valuable lesson on the order of declared variables, specifically that reassigning variables values doesn’t change the code that came before it.

An example of a problem that I am proud of solving was learning how to count backwards using a for loop. My initial code was:

const countdown = [‘1’, ’2’, ‘3’, ‘blastoff’ ]

function printCountdown(countdown) {

for (let i = countdown.length; i > 0; i—) {

console.log(countdown[I] + ‘!’)

}}

Initially I thought that this code was spot on. Obviously it didn’t work, so I asked my classmate where I went wrong. He told me that the problem was in the ‘i = countdown.length;’ section of the for loop, but he didn’t tell me what was wrong with it. I sat there for 5 minutes and thought about it. I then had the realisation that when you call something from an array, you start counting from zero. So the string that says ‘1’ would be [0] in the array, ‘2’ would be [1] in the array etc, but countdown.length gives me the actual amount of strings in the array, which in this case is 4. So with the code I have now, Im trying to call [4] which doesn’t exist in the array. So then I thought I must just have to -1 from the countdown length, which got me to my final code of:

const countdown = [‘1’, ’2’, ‘3’, ‘blastoff’ ]

function printCountdown(countdown) {

for (let i = countdown.length - 1; i > 0; i—) {

console.log(countdown[I] + ‘!’)

}}

It was very satisfying to figure this out on my own, and I’m grateful to my classmate for not giving me the answer right away and allowing me to figure it out.

Reflection on my confidence using certain problem-solving techniques

Pseudocode:

I haven’t had a proper go at using pseudocode yet, but I can see how it would be useful. I think once I start on bigger projects, it will end up being an essential part of my tool-kit.

Trying something:

This is what I mainly do, I think of something that might solve a problem and I use console.log to see if it gives me the desired result.

Rubber ducky method:

This is something I haven’t really tried when solving a problem, but it is similar to the process of reflecting after completing a task wherein you have to explain something as if your teaching it to someone else which I find to be very helpful to identify areas of a subject that I might not be so adept at.

Reading error messages:

This is relatively easy as error messages tend to give you a reason for the error and will show you where in the code the error has occurred. I’d say I’m pretty confident with this.

Googling:

Googling is one of my go to’s. I will usually do this before asking a classmate for help.

Asking your peers and coaches for help:

I have been asking my peers for help a lot throughout the intro to JavaScript and they have been immensely helpful in helping me to understand certain concepts. I’d say I am pretty good at doing this, I just hope I’m not getting too annoying.

I find myself wanting to ask a lot of questions, so sometimes I will be reluctant to ask a question because I feel like I am asking too much.
